   If the file came from another editor that saved it as UTF-16, VS Code may keep that encoding when you save until you explicitly “Save with Encoding” → “UTF-8”.

3. **A setting overrides the default**  
   If `files.encoding` is set to something like `utf16le`, then files can be saved as UTF-16.

**To fix and avoid UTF-16 for `requirements.txt`:**

- Open `requirements.txt`.

## How to add a column to `image_collection.csv`

The columns for `image_collection.csv` are defined in **`image_collection_metadata.csv`**. The program uses that file to:

- Create new collections (File → New)
- Add folders to existing collections (Add Folders)
- Open existing CSVs (missing columns get the default from metadata)

### 1. Add a row in `image_collection_metadata.csv`

Each column is one row with these fields (header row is the first line):

| Field       | Meaning |
|------------|---------|
| **col_name** | Column name (no spaces; use underscore). |
| **col_type** | `int`, `float`, `str`, or `any`. |
| **default**  | Value when no EXIF/source is used (e.g. `0`, `na`, ` `, or empty). |
| **load_func**| How to fill from EXIF: `get_key`, `get_lat_lon`, `get_key_value`, or leave empty to use **default** only. |
| **exif_tags**| EXIF tag name(s) for that loader (e.g. `Image Make` or a JSON array of alternatives). Leave empty if you only use **default**. |
| **notes**    | Optional description. |

**Examples:**

- **Column filled only by you later** (like `notes`, `img_tags`):  
  Leave `load_func` and `exif_tags` empty; set `default` as you like (e.g. ` ` or empty).

- **Column from a single EXIF tag:**  
  `load_func` = `get_key`, `exif_tags` = tag name (e.g. `EXIF ExposureTime`).

- **Column from one of several possible EXIF tags:**  
  `load_func` = `get_key`, `exif_tags` = JSON array, e.g. `["EXIF DateTimeOriginal","Image DateTime"]`.

### 2. Example added in your project

A sample column was added at the end of `image_collection_metadata.csv`:

```csv
rating,str,,,,optional 1-5 or custom rating - leave default blank to fill later
```

So:

- **New collections** will have a `rating` column (default blank).
- **Existing `image_collection.csv` files** will get a `rating` column when you open them (with the default value).
- You can edit that row (e.g. change `rating` to your column name, or delete the line if you don’t want it).

### 3. No code changes needed

The table and EXIF loader already use the metadata. As long as the new column is defined in `image_collection_metadata.csv` with the right `col_name`, `col_type`, `default`, and optionally `load_func`/`exif_tags`, it will appear in the CSV the program creates and opens. If you need the new column to show up in a specific part of the UI (e.g. gallery, tree, or export), that’s the only place you might add code; for “just another column in the CSV,” the metadata change is enough.
