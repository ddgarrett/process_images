## Thumbnail in the click popup (info window)

**Yes.** `info_window` is documented as **HTML** for the Maps JavaScript `InfoWindow`. You already pass HTML with links in `pi_action_map.py` (`info_window=v`). You can extend that string with a normal image tag, for example a line like:

`<img src="https://your-site/path/to/thumb.jpg" width="120" height="120" alt="">`

plus your existing `<a href="...">...</a>`.

Practical notes:

- The map page is loaded from a **local file** (`temp_map.html`). The browser will still load `https://...` images in the info window; avoid `http://` images if the page is treated as a secure context or you hit mixed-content rules.
- The `src` must be something the **browser** can fetch (public URL). A `file://` path to a local JPEG usually will **not** work from that HTML page the way a hosted URL does.
- Escape double quotes inside your HTML if you build strings in Python (`\"` inside the outer string, or use single-quoted attributes in HTML).

So: **thumbnail in addition to (or instead of) the link text in the popup** is supported by the platform and by gmplot’s `info_window` string.

## Thumbnail *on the map* instead of (or next to) the pin

**Not through gmplot’s public API.** In gmplot’s implementation, the marker’s `icon` in the generated `google.maps.Marker({...})` comes from their **color pin** system (fixed palette / embedded icon URLs), not from an arbitrary per-photo image URL. The `marker()` method only documents things like `color`, `title`, `label`, `info_window`, `draggable`, `precision` — there is no supported `icon_url` / `thumbnail` argument.

The **underlying** Maps API *does* allow a custom marker `icon` with a `url` and `scaledSize` (classic `google.maps.Marker`). To use that with your stack you would need something outside vanilla gmplot, for example:

- **Patch or fork gmplot** so `_Marker` / `_RawMarker` can emit `icon: { url: '...', scaledSize: ... }`, or  
- **Post-process** the HTML after `draw()`, or  
- **Generate the map** with your own small JS template instead of gmplot.

Google is steering new work toward **advanced markers** (`AdvancedMarkerElement`), but classic markers with image URLs are still a common pattern; gmplot just does not expose it.

## Short recommendation

- **Easiest win:** put the thumbnail **inside `info_window`** next to your existing link.  
- **Pin replaced by a photo on the map:** possible with **Google’s** API, **not** with **stock gmplot** without extra work (fork, post-process, or custom HTML/JS).

**No.** gmplot (including **1.4.1** and current upstream) is built around the classic **`google.maps.Marker`** API. Its marker code is generated roughly as `new google.maps.Marker({ position: ..., icon: ..., map: map })` (see the library’s `_RawMarker` / marker pipeline on [gmplot’s GitHub](https://github.com/gmplot/gmplot)).

**Advanced markers** (`AdvancedMarkerElement`, the marker library, and typically a **`mapId`** / vector map setup) are a different API. gmplot does not load the advanced marker library or emit `AdvancedMarkerElement` in the HTML it writes.

If you need advanced markers, you would use the Maps JavaScript API directly (or another wrapper), not stock gmplot.

is there another way to use advanced markers from python to generate an html map i javascript, like what gmplot does?

**Cursor**

There is no widely used **“gmplot but for AdvancedMarkerElement”** library. Advanced markers live in the **browser** (`google.maps.marker.AdvancedMarkerElement` via `importLibrary("marker")`), while most Python “map” packages either emit **classic** `google.maps.Marker` (like gmplot) or target **Leaflet/Mapbox** (folium, ipyleaflet, pydeck), not Google’s advanced marker stack.

**Practical approaches (same idea as gmplot: Python writes HTML + JS):**

1. **Template + JSON (most common)**  
   Use Python only to **build data** (center, zoom, list of `{lat, lng, title, htmlContent, ...}`) and **render** a small HTML file (Jinja2, string formatting, or any templater). The template contains the official JS: load the Maps script with `libraries=marker`, `await google.maps.importLibrary("maps")` and `importLibrary("marker")`, create `Map` with a **`mapId`**, then create `AdvancedMarkerElement` instances. This is the direct analogue of what gmplot does, except you own the template.

2. **Jupyter**  
   Same HTML/JS, displayed with `IPython.display.HTML` (or an `iframe` pointing at a generated file). Still not a separate “advanced marker” Python package.

3. **Google’s Python client** (`google-maps-services-python`)  
   Helps with **web services** (geocoding, directions, etc.), **not** generating client-side advanced marker HTML.

4. **Web app stack**  
   If the output is a real app, **React/Vue + `@vis.gl/react-google-maps`** (or similar) wraps advanced markers; Python might only serve API data, not the map HTML.

**Important requirement:** Advanced markers expect a **vector map with a Map ID** (`mapId` in `Map` options) and the right **API enablement/billing** in Cloud Console. That is separate from Python; your generator just has to put the right IDs and script URL into the HTML.

**Bottom line:** Yes, you can get the **same outcome as gmplot** (Python produces a standalone `.html` that opens in a browser) by **templating JavaScript** yourself or with a thin helper; there is not a mature drop-in Python package that matches gmplot’s API but targets advanced markers.

    _IMG_SRC_RE = re.compile(
        r'<img\b[^>]*\bsrc\s*=\s*([\'"])(?P<src>.*?)\1',
        re.IGNORECASE | re.DOTALL,
    )

    @classmethod
    def _get_figure_image_src(cls, anchor_html: str):
        '''First <img src="..."> after the anchor and before <figcaption>, if any.'''
        if not anchor_html:
            return None
        low = anchor_html.lower()
        fc = low.find('<figcaption>')
        head = anchor_html[:fc] if fc >= 0 else anchor_html
        m = cls._IMG_SRC_RE.search(head)
        if not m:
            return None
        src = m.group('src').strip()
        return src or None
