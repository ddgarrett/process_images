## What `all_maps.html` should do

Goal: **one world map** with every **unique** `garrettblog.com/...#photo.jpg` entry once, with the same info-window HTML (thumbnail + date/caption link) that `pi_action_map` produces.

Recommended algorithm:

1. **Scan** each map HTML under `/Users/douglasgarrett/Documents/projects/ddgarrett.github.io` that contains `new google.maps.InfoWindow` and `garrettblog.com`.
2. **Parse** each `info_marker_N` / `info_window_N` pair (same regex approach as `upgrade_map_info_thumbnails.py`: `content: '...'` with JS single-quote escaping).
3. **Split** multi-photo windows on `<BR>` and extract the primary `href` per block.
4. **Deduplicate** by full URL (`https://www.garrettblog.com/2023/04#PXL_....jpg`). Keep the first seen lat/lng; if a later copy has `<img>` and the stored one does not, prefer the richer HTML (matches your thumbnail upgrade behavior).
5. **Merge co-located photos** exactly like `pi_action_map.py`: group by rounded `(lat, lon)`, join blocks with `<BR>`, increment heatmap weight per photo at that point.
6. **Emit** a new gmplot-style HTML file (markers + `HeatmapLayer`), with zoom/center over all points (e.g. global zoom ~2-3 or centroid of all coords).

You already have building blocks in `process_images`:

- Marker HTML shape: `pi_action_map.py` (`_generate_map`)
- Parsing/upgrading saved maps: `upgrade_map_info_thumbnails.py`

A dedicated script (e.g. `merge_maps.py` in either repo) is cleaner than hand-merging 2,800+ entries.
