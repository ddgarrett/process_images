The actual rules are in **`_status_for_row`** (order matters: poor quality and duplicates are checked before score bands):

```462:479:/Users/douglasgarrett/Documents/projects/image_analysis_lib/image_analysis_lib/duplicates.py
def _status_for_row(
    relative_path: str,
    score: float | None,
    duplicate_to_keeper: dict[str, str],
    *,
    poor_quality_threshold: float,
) -> tuple[str, str]:
    """Return (status, dup_photo). dup_photo only set when status is 'dup'."""
    if score is not None and score < poor_quality_threshold:
        return "poor quality", ""
    if relative_path in duplicate_to_keeper:
        return "dup", duplicate_to_keeper[relative_path]
    if score is not None:
        if score > 6:
            return "best", ""
        if score > 5:
            return "good", ""
    return "TBD", ""
```

So the possible **`status`** values are exactly: **`poor quality`**, **`dup`**, **`best`**, **`good`**, and **`TBD`** (the README in that repo lists the same set). Edge cases: missing/invalid `musiq_score` skips the poor-quality and score bands and can land on **`dup`** or **`TBD`**.

Unit tests for this behavior live in `tests/test_duplicates.py` under tests for `_status_for_row`.
