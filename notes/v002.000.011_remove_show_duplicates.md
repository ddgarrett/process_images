## History rewrite vs. revert

- **`git revert`** (new commit that undoes an old one): safe for shared branches; may conflict as above.
- **Dropping the commit with rebase/filter** so it never existed: harder operationally (force-push, coordination with clones) and **still** conflicts with the same overlapping files if you replay later commits unless you resolve each step.
