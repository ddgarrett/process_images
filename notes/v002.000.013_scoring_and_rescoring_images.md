## 1) Values written by dedup to `image_scores_and_status.csv`

In `image_analysis_lib`, `status` is set in `_status_for_row` to one of:

- `poor quality`
- `dup`
- `best`
- `good`
- `TBD`

```472:490:/Users/douglasgarrett/Documents/projects/image_analysis_lib/image_analysis_lib/duplicates.py
def _status_for_row(
    relative_path: str,
    score: float | None,
    duplicate_to_keeper: dict[str, str],
    *,
    poor_quality_threshold: float,
) -> tuple[str, str]:
    if score is not None and score < poor_quality_threshold:
        return "poor quality", ""
    if relative_path in duplicate_to_keeper:
        return "dup", duplicate_to_keeper[relative_path]
    if score is not None:
        if score > 6:
            return "best", ""
        if score > 5:
            return "good", ""
    return "TBD", ""
```

## 2) How dedup decides `status = "dup"` (cosine logic)

`duplicate_to_keeper` is built in `find_duplicates_by_score`:

- Rows are sorted by MUSIQ score descending.
- Only rows with `score >= poor_quality_threshold` are considered for duplicate matching.
- Comparisons are within same folder only.
- Optional GPS gate can skip comparisons if distance exceeds radius.
- If cosine similarity `>= min_similarity_threshold` (default `0.65`), lower-scoring image becomes duplicate of higher-scoring keeper.

`image_collection.csv` stores raw values like `img_status='tbd'` and `rvw_lvl='4'`, and then `ImageCollection.translate_status_lvl()` converts that for display.

```20:33:/Users/douglasgarrett/Documents/projects/process_images/image_collection.py
_tbd_lvl_translate = ("Reject?", "Bad?", "Dup?", "Good?", "Best?", "???")

@staticmethod
def translate_status_lvl(status, level):
    """translate status and level into something more human friendly"""
    if status == "tbd":
        level = _tbd_lvl_translate[int(level)]
    else:
        level = ""

    return [status, level]
```

So for `img_status='tbd'`, the displayed second value is:

- `rvw_lvl='0'` -> `Reject?`
- `rvw_lvl='1'` -> `Bad?`
- `rvw_lvl='2'` -> `Dup?`
- `rvw_lvl='3'` -> `Good?`
- `rvw_lvl='4'` -> `Best?`
- `rvw_lvl='5'` -> `???` (fallback slot)

And yes, this is how you get a display like `tbd, Good?` (that corresponds to `img_status='tbd'` + `rvw_lvl='3'`).
