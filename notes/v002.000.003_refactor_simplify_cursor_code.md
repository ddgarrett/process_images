Here’s what changed.

### Behavior
- **`image_scores_and_status.csv`** rows are keyed only by **`(file_location, file_name)`**, exactly as stored in the CSV (no slashes, basename, or `root_dir` resolution).
- The collection is still updated with `k = (r["file_location"], r["file_name"])`, so a score applies when those two values match **as-is**.
- The CSV must include columns **`musiq_score`**, **`file_location`**, and **`file_name`**; otherwise the loader skips applying scores (same idea as the existing `musiq_score` check).

### Removed code
- **`_norm_collection_subdir`**, **`_key_from_path_relative_root`**, and **`_musiq_key_from_scores_row`** were only used for the old path-based matching, so they were deleted.

If any cell differs only by trivial things (extra spaces, `None` vs `""`), those rows still won’t match until the data is aligned—by design, given “equal” with no normalization.
