## Staying above the main app but not above other apps

`keep_on_top=True` maps to Tk’s **global topmost** behavior, so the dialog tends to stay above **other applications**, not just your main window. That is the tradeoff your notes describe.

To prefer **“above the main Process Images window”** without insisting on **“above every app”**, the usual approach on Tk is **`wm transient`** for the Properties `Toplevel`, with the main window’s root as parent: after `finalize=True`, something like calling Tk’s `transient` on the properties window with `c.window.TKroot` as the master. That ties the dialog to the main app’s window group for stacking; it is **not** the same as system-wide topmost.

Concrete recipe people use:

1. After creating and finalizing the Properties window, set transient to `c.window.TKroot`.
2. **Drop or avoid** `keep_on_top=True` on Properties if you want other apps (browser, etc.) to cover it when you switch focus; rely on `transient` (and often `modal=True`) so Properties still stays correctly stacked **relative to** the main window.

Exact stacking rules vary a bit by OS (macOS vs Windows), so you would verify on your machine after swapping `keep_on_top` for `transient`.
